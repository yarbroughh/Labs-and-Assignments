Practice Form with All Input Field Options

The following are common options. This is to provide practice for several common input fields that are available. I'll then apply to my main project.

  • Text is a standard text input field
  • Password hides input characters
  • Number is for numerical input, so it has built-in number controls
  • Email for email addresses
  • Date for selecting a date
  • Time for selecting a time
  • Datetime-local for selecting both date and time
  • Color for selecting a color
  • File for flie uploads
  • Checkbox for multiple choice that can be checked or left unchecked
  • Radio for when you have a group of options that requires one choice-It's important to give each option the same Name attribute within the input element
  • Submit is a button to submit the form
  • Button is alos a button but is generic so can be used to link etc. and doesn't submit the form
  • Reset is a button that resets all form fields
  • Image uses an image as the submit button
  • There are several others but are more specialized so I can look those up on an as-needed basis

Input Tag Attributes

The following list shows the different attributes that can be included within an input tag. The only required attribute is type, but when you want to submit the entered date, the name attribute is needed.>

  • Type is required and that's the Input Field Option from the list above. It defines the input element's appearance, behavior, and how data is entered
  • Name is rquired for form submissions. It assigns a name to hte input element and when the form is submitted the browser sends the name and its value to the server
  • ID is a unique identifier within the HTML document and is used wiht the for attribute in Label tags to assosciate a label with the input. JavaScript uses the ID
  • Value pre-fills the input. For example in checkbox and radio buttons it specifies the value that will be submitted
  • Placeholder displays a hint for expected input such as "Enter your Email Address" or "e.g., 555-555-1212"
  • Required makes the field mandatory
  • Disabled makes it uneditable and unclickable
  • Readonly makes it read-only so the user can see the value but can't modify it
  • maxlength sets max number of characters
  • min/max is used for number and date/time inputs to specify the minimum or maximum values allowed. These are entered separately as min="#" and max="#"
  • Step is also for number inputs and defines the interval between allowed values
  • List connects the input field to a datalist element which provides a list of suggested values
  • Pattern defines a regular expression that the input value must match
  • Autocomplete controls whether the broswer should try to autocomplete based on previous entries
